= Using subscriptions
include::modules/common-attributes.adoc[]
:context: serverless-subs

toc::[]

After events have been sent to a channel from an event source or producer, these events can be sent to multiple Knative services, or other sinks, by using a subscription.

image::serverless-event-channel-workflow.png[Channel workflow overview]

If a subscriber rejects an event, there are no re-delivery attempts by default. Developers can configure xref:../../serverless/event_workflows/serverless-event-delivery.adoc#serverless-event-delivery[re-delivery attempts] by modifying the `delivery` spec in a `Subscription` object. Cluster administrators can also configure re-delivery attempts globally by modifying the `config-br-delivery` config map.
